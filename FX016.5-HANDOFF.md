## What Was Built

### 1. Opening Price Model (`lib/opening-price-model.ts`)

New deterministic pricing model replacing the naive flat-base approach.

**Inputs**: projection, position, threshold (TOP_3/TOP_5/TOP_10), status
**Output**: YES probability in [0.05, 0.95]

**Logic**:
1. `projectionToRank()` — maps projection onto positional rank using elite/floor/poolSize per position
2. `rankToProb()` — logistic sigmoid centered on threshold cutoff (steepness varies by threshold)
3. Status multipliers: ACTIVE=1.0, QUESTIONABLE=0.80, DOUBTFUL=0.45, OUT=0.05
4. Final result clamped to [0.05, 0.95]

**Position profiles**:
- QB: elite=28, floor=12, poolSize=18
- RB: elite=24, floor=8,  poolSize=30
- WR: elite=25, floor=7,  poolSize=40
- TE: elite=18, floor=5,  poolSize=15

**Examples**:
- Josh Allen (QB, 27.4 proj) Top 3 → rank ~1 → ~85% YES
- Travis Kelce (TE, 15.2 proj) Top 3 → rank ~4 → ~43% YES
- Borderline WR (9.5 proj) Top 5 → rank ~28 → ~5% YES

Also exports `calcInitialPools()` which derives AMM pool values from opening probability.

### 3. Bloomberg Terminal Design Components (`components/ui/terminal-panel.tsx`)

New set of financial-terminal-style UI primitives:

| Component | Purpose |
|-----------|---------|
| `TerminalPanel` | Dark bordered panel with monospace label header |
| `TerminalHeader` | Dense section header with amber `▸ LABEL` style |
| `TerminalDivider` | Hairline section divider with optional label |
| `PriceCell` | Monospace price in cents (`68¢`) with up/down color |
| `ChangeCell` | Colored Δ% with trend arrow (`+4.2%` / `-1.8%`) |
| `VolumeCell` | Right-aligned volume formatted as `12.5K` |
| `QuoteRow` | Full market quote row (instrument · YES · NO · Δ% · VOL) |
| `TapeRow` | Exchange tape line: `10:42:18  BUY  YES  Jefferson  T5  WR  68¢` |
| `MarketHeatCell` | Color-coded block showing YES% (green→amber→red) |

All use `font-mono`, `tabular-nums`, tight spacing — Bloomberg aesthetic.

### 4. Pixel Player Avatars (`components/ui/pixel-avatar.tsx`)

Original 8-bit stylized football player avatars. No NFL IP used.

**Features**:
- 16×16 SVG pixel grid scaled to xs/sm/md/lg/xl sizes
- Position-specific stances: QB throwing · RB running · WR catching (arms up) · TE set position (wider, arms out)
- Team-color uniform (primary/secondary from `getTeamColors()`)
- Deterministic variation per playerId — same player always renders identically
- Skin tone + hair color from seeded palette (generic, no ethnicity identification)
- Gloves, shoulder pads, ball tucked/in-flight per position

**Usage**:
```tsx
<PixelPlayerAvatar
  playerId="p_josh_allen"
  playerName="Josh Allen"
  team="BUF"
  position="QB"
  size="lg"
/>
```

### 5. Wall Street Market Board (`app/markets/board/page.tsx`)

New dedicated market board page at `/markets/board`.

**Features**:
- Full quote table: instrument · YES¢ · NO¢ · Δ% · VOL
- Live row flashing (green/red) when prices change via SSE
- 7 sort modes: volume · gainers · losers · locking · OI · YES↑ · YES↓
- Filters: position (QB/RB/WR/TE) · threshold (T3/T5/T10) · status (OPEN/LOCKED/SETTLED)
- Search by player name or team
- Countdown strip for markets locking within 6 hours
- Row selection highlight
- Mobile-responsive (filters collapse, column headers hidden on xs)
- `LiveBadge` indicator

### 6. Exchange Tape Feed (`components/ui/exchange-feed.tsx`)

## Known Issues

- Pixel avatars display at small sizes (xs/sm) — detail is limited by pixel grid; use md+ for meaningful rendering
- `openingPrice` in DB reflects model price at seed time; if user re-seeds, market movers reset
- Gainers/Losers sections are empty until demo trades move prices past opening
- Board page flash detection requires SSE to be active; polling fallback won't trigger row flashes (prices only update every 12s with no diffing)
